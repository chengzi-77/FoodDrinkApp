# 食光营养助手项目开发指南

## 1. 项目概述

本项目是一个基于 .NET MAUI 的跨平台移动应用，应用名称为“食光营养助手”。项目主题符合需求文档中的 Food and Drink 要求，核心功能是记录食品和饮品、查看营养信息，并通过移动设备硬件增强用户体验。

应用主要支持：

- 食品和饮品列表浏览
- 搜索食品、饮品、分类或标签
- 添加新的食品或饮品记录
- 查看营养详情
- 拍摄食品照片
- 获取用餐位置，包括国家、城市、区域和经纬度
- 文字转语音朗读营养摘要或帮助内容
- 停止朗读，避免离开页面后继续播放
- 震动与触觉反馈
- 大字体模式、深色/浅色主题和屏幕阅读器播报

## 4. 主要文件解释

### App.xaml / App.xaml.cs

`App.xaml` 加载全局资源字典，包括颜色和控件样式。

`App.xaml.cs` 是应用入口之一，创建主窗口并加载 `AppShell`：

```csharp
return new Window(new AppShell());
```

### AppShell.xaml / AppShell.xaml.cs

`AppShell.xaml` 定义应用导航结构。当前使用底部 TabBar：

- 食品
- 硬件
- 设置

`AppShell.xaml.cs` 注册详情页和添加页路由：

```csharp
Routing.RegisterRoute(nameof(AddItemPage), typeof(AddItemPage));
Routing.RegisterRoute(nameof(FoodDetailPage), typeof(FoodDetailPage));
```

这样首页可以通过 Shell 跳转到添加页和详情页。

### MainPage.xaml / MainPage.xaml.cs

首页负责展示食品和饮品列表。

主要控件：

- `SearchBar`：搜索食品、饮品、分类或标签
- `CollectionView`：展示食品卡片
- `Button`：跳转添加页或详情页
- `RefreshView`：下拉刷新列表

核心逻辑：

```csharp
FoodCollection.ItemsSource = FoodCatalogService.Search(query);
```

首页不直接维护数据，而是通过 `FoodCatalogService` 获取列表，代码更清晰。

### AddItemPage.xaml / AddItemPage.xaml.cs

添加页负责让用户新增食品或饮品记录。

表单字段包括：

- 名称
- 分类
- 描述
- 热量
- 蛋白质
- 碳水
- 脂肪
- 过敏提示

验证逻辑在 `ValidateForm` 中：

- 名称不能为空
- 分类必须选择
- 描述不能为空
- 热量、蛋白质、碳水、脂肪必须是非负数字

如果验证失败，会显示错误提示面板，并触发震动反馈：

```csharp
ShowValidation(validationMessage);
Vibration.Default.Vibrate(TimeSpan.FromMilliseconds(250));
```

保存成功后会调用：

```csharp
FoodCatalogService.Add(new FoodItem { ... });
```

### FoodDetailPage.xaml / FoodDetailPage.xaml.cs

详情页展示某一条食品或饮品的营养信息。

展示内容包括：

- 名称
- 分类
- 热量
- 蛋白质、碳水、脂肪
- 描述
- 过敏提示

详情页支持：

- 朗读营养摘要
- 停止朗读
- 震动提醒

朗读使用 `SpeechService`：

```csharp
await SpeechService.SpeakChineseAsync(currentItem.AccessibleSummary);
```

离开页面时会自动停止朗读：

```csharp
protected override void OnDisappearing()
{
    SpeechService.Stop();
    base.OnDisappearing();
}
```

这避免了退出页面后还继续播放语音的问题。

### HardwarePage.xaml / HardwarePage.xaml.cs

硬件页集中展示移动设备硬件能力。

已实现硬件功能：

| 功能 | API |
|---|---|
| 拍照 | `MediaPicker.Default.CapturePhotoAsync()` |
| 定位 | `Geolocation.Default.GetLocationAsync()` |
| 地址解析 | `Geocoding.Default.GetPlacemarksAsync()` |
| 文字转语音 | `TextToSpeech.Default.SpeakAsync()`，封装在 `SpeechService` 中 |
| 震动 | `Vibration.Default.Vibrate()` |
| 触觉反馈 | `HapticFeedback.Default.Perform()` |

定位逻辑：

```csharp
var location = await Geolocation.Default.GetLocationAsync(request);
CoordinateLabel.Text = $"纬度 {location.Latitude:F5}，经度 {location.Longitude:F5}";
LocationLabel.Text = await BuildAddressTextAsync(location);
```

地址解析优先使用系统反向地理编码。如果模拟器不返回国家和城市，会使用兜底逻辑。例如 Google 模拟器默认坐标 `37.422, -122.084` 会显示：

```text
美国 / 加利福尼亚州 / 山景城
```

触觉反馈为了便于录屏验证，增加了计数：

```csharp
feedbackTestCount++;
FeedbackCountLabel.Text = $"触觉反馈测试次数：{feedbackTestCount}";
```

### SettingsPage.xaml / SettingsPage.xaml.cs

设置页负责无障碍相关功能。

当前支持：

- 跟随系统主题
- 浅色主题
- 深色主题
- 大字体模式

大字体模式通过 `AccessibilityService` 实现。打开开关后，当前页面文字立即放大；切换到其他页面后，在页面 `OnAppearing` 中继续应用字体缩放。

```csharp
AccessibilityService.LargeTextEnabled = e.Value;
AccessibilityService.ApplyFontScale(this);
```

### Models/FoodItem.cs

`FoodItem` 是食品/饮品数据模型。

主要属性：

- `Name`
- `Category`
- `Description`
- `Calories`
- `Protein`
- `Carbs`
- `Fat`
- `AllergyNote`
- `Tags`

它还提供用于界面显示和语音朗读的计算属性：

```csharp
public string CaloriesLabel => $"{Calories} 千卡";
public string MacroSummary => $"蛋白质 {Protein}克，碳水 {Carbs}克，脂肪 {Fat}克";
public string AccessibleSummary => $"{Name}。{Category}。{Calories} 千卡。{MacroSummary}。{AllergyNote}";
```

### Services/FoodCatalogService.cs

食品数据服务，负责从 mockapi.io 或本地兜底数据读取食品信息：

- 优先请求 mockapi.io REST API
- 网络不可用或没有配置 API 时使用本地兜底数据
- 搜索食品
- 根据 ID 获取详情
- 添加新食品记录

搜索会匹配：

- 名称
- 分类
- 描述
- 标签

mockapi.io 的接口地址配置在：

```text
FoodDrinkApp/Services/MockApiConfig.cs
```

将 `EndpointUrl` 改成 mockapi.io 生成的 Resource 地址即可，例如：

```csharp
public const string EndpointUrl = "https://682xxxx.mockapi.io/api/v1/foods";
```

详细配置步骤见项目根目录下的 `mockapi配置说明.md`。

### Services/SpeechService.cs

语音朗读服务，负责统一处理 Text-to-speech。

功能包括：

- 开始新朗读前停止旧朗读
- 支持停止朗读
- 优先选择中文语音
- 设置音量和音高，让声音稍微更自然

核心代码：

```csharp
var options = new SpeechOptions
{
    Volume = 0.9f,
    Pitch = 1.08f,
    Locale = await FindChineseLocaleAsync()
};
```

注意：语音是否“好听”取决于设备系统安装的 TTS 语音包。真机通常比 Android 模拟器自然。

### Services/AccessibilityService.cs

无障碍字体缩放服务。

它会遍历当前页面上的 Label、Button、Entry、Editor、Picker、SearchBar 等控件，并根据大字体开关放大或恢复字体。

### Platforms/Android/AndroidManifest.xml

Android 权限配置文件。

当前包含：

```xml
<uses-permission android:name="android.permission.CAMERA" />
<uses-permission android:name="android.permission.ACCESS_COARSE_LOCATION" />
<uses-permission android:name="android.permission.ACCESS_FINE_LOCATION" />
<uses-permission android:name="android.permission.VIBRATE" />
```

这些权限对应相机、定位和震动功能。

### Resources/Styles/Styles.xaml

全局控件样式。

当前统一了：

- Button
- Entry
- Editor
- Picker
- SearchBar
- Label
- Shell TabBar

视觉风格采用暖奶油背景、番茄红、烤橙和罗勒绿，让应用更符合食品饮品主题。

## 5. 构建与运行

由于当前项目路径包含中文和 `&`，Android 构建工具可能会出现路径问题。项目根目录下的 `Directory.Build.props` 已将构建输出改到：

```text
C:\MauiBuild\FoodDrinkApp\
```

Windows 构建：

```powershell
dotnet build .\FoodDrinkApp\FoodDrinkApp.csproj -f net9.0-windows10.0.19041.0 --no-incremental
```

Android 构建：

```powershell
dotnet build .\FoodDrinkApp\FoodDrinkApp.csproj -f net9.0-android --no-incremental
```

Android 启动运行：

```powershell
dotnet build .\FoodDrinkApp\FoodDrinkApp.csproj -f net9.0-android -t:Run
```

如果 Visual Studio 提示“无法为 Android 启动此项目”，通常是没有在线设备。需要先启动 Android 模拟器或连接真机。

## 6. Screencast 演示建议

建议录屏按以下顺序：

1. 介绍应用主题：Food and Drink，应用名“食光营养助手”。
2. 展示首页 UI、搜索框、食品卡片和底部导航。
3. 搜索一个食品，例如“饮品”或“早餐”。
4. 打开详情页，展示营养信息。
5. 点击“朗读摘要”，再点击“停止朗读”，说明离开页面会自动停止。
6. 点击“震动提醒”，说明使用了 Vibration 和 Haptic feedback。
7. 打开添加页，故意留空或输入非法数字，展示验证和错误提示。
8. 正确添加一条食品记录，返回首页展示新增内容。
9. 打开硬件页，演示拍照。
10. 演示定位，说明会显示国家、城市、区域和经纬度。
11. 演示朗读帮助和停止朗读。
12. 演示触觉反馈，说明计数变化用于录屏验证。
13. 打开设置页，演示深色/浅色主题和大字体模式。
14. 展示代码结构：Models、Services、页面 XAML、AndroidManifest 权限。
15. 展示 Android 和 Windows 构建或运行结果。
16. 展示 GitHub 提交记录和 README。

## 7. 仍需注意的事项

- GitHub 使用占 5%，需要多次提交，不能最后一次性提交。
- Screencast 是主要评分来源，必须清楚讲解每个评分点。
- 如果定位在模拟器上无法显示真实城市，可以在模拟器 Extended Controls 中设置 Location，或使用真机测试。
- Text-to-speech 的音色依赖系统语音包；如果要更自然，建议在真机安装或选择更好的中文 TTS 引擎。
